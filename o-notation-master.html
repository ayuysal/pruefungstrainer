📈 O-Notation Grundlagen

Obere Schranke für die Laufzeit eines Algorithmus.

  • O(1) – Konstant
  • O(log n) – Logarithmisch
  • O(n) – Linear
  • O(n log n) – Linearithmisch
  • O(n²) – Quadratisch
  • O(n³) – Kubisch
  • O(2ⁿ) – Exponentiell
  • O(n!) – Faktoriell

📊 Wachstumshierarchie

Von langsam nach schnell wachsend:

1 < log n < √n < n < n log n < n² < n³ < 2ⁿ < n!

Ω(f) = untere Schranke
Θ(f) = enge Schranke (obere UND untere)
O(f) = obere Schranke

🧪 Best / Average / Worst Case

  • Best Case: Günstigste Eingabe
  • Average Case: Erwartungswert über alle Eingaben
  • Worst Case: Ungünstigste Eingabe (O-Notation)

Beispiel QuickSort:
Best: O(n log n), Avg: O(n log n), Worst: O(n²)

⚖️ Master-Theorem

Für T(n) = a · T(n/b) + f(n):

Fall 1: f(n)=O(nlogb(a)-ε)
→ T(n) = Θ(nlogb(a))
Fall 2: f(n)=Θ(nlogb(a))
→ T(n) = Θ(nlogb(a) · log n)
Fall 3: f(n)=Ω(nlogb(a)+ε)
→ T(n) = Θ(f(n))

🔍 Bekannte Komplexitäten

  • Binary Search: O(log n)
  • Linear Search: O(n)
  • MergeSort: O(n log n)
  • QuickSort: O(n²) worst / O(n log n) avg
  • BubbleSort: O(n²)
  • Matrix-Multiplikation: O(n³)

🧩 P vs NP

  • P: In Polynomialzeit loesbar
  • NP: In Polynomialzeit verifizierbar
  • NP-vollstaendig: Härteste Probleme in NP
  • Offenes Problem: P = NP?

Bsp: Traveling Salesman (NP-vollst.), Sortieren (P), SAT (NP-vollst.)

💡 Merke: Master-Theorem

Berechne immer zuerst logb(a) und vergleiche mit dem Exponenten von f(n). Ist f(n) polynomiell kleiner → Fall 1. Gleich → Fall 2. Größer → Fall 3.

🎯 Prüfungstipp

Master-Theorem ist Pflicht! Typische Aufgabe: T(n)=4T(n/2)+n → log2(4)=2, f(n)=n=O(n2-ε) → Fall 1 → Θ(n²). Rechne logb(a) IMMER zuerst!

🧩 Standardbeispiele

MergeSort: T(n)=2T(n/2)+n → Fall 2 → Θ(n log n)
Binary Search: T(n)=T(n/2)+1 → Fall 2 → Θ(log n)
Strassen: T(n)=7T(n/2)+n² → Fall 1 → Θ(nlog27)

⚠️ Häufige Fehler

1. logb(a) falsch berechnet
2. Vergessen, dass f(n)=n² bedeutet c=2
3. Verwechslung: O ist obere, Ω untere Schranke
4. Master-Theorem gilt NUR für T(n)=aT(n/b)+f(n)
