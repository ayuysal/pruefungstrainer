TCP vs. UDP & Routing

Transportprotokolle, Handshake, Portnummern und Routing-Protokolle – interaktiv lernen und üben

📚

Referenz

🔁 TCP – Transmission Control Protocol

  • Verbindungsorientiert – Verbindungsaufbau vor Datentransfer
  • Zuverlässig – Bestätigungen (ACK), Neuübertragung bei Verlust
  • Flusskontrolle – Sliding-Window-Verfahren
  • Reihenfolge – Sequenznummern garantieren korrekte Reihenfolge
  • Header: 20–60 Bytes (größer als UDP)
  • Einsatz: HTTP, HTTPS, FTP, SSH, SMTP, POP3, IMAP

⚡ UDP – User Datagram Protocol

  • Verbindungslos – kein Verbindungsaufbau nötig
  • Unzuverlässig – keine Bestätigungen, keine Neuübertragung
  • Keine Flusskontrolle – Sender sendet ohne Rücksicht
  • Keine Reihenfolge – Pakete können vertauscht ankommen
  • Header: 8 Bytes (minimal)
  • Einsatz: DNS, DHCP, VoIP, Streaming, Gaming, TFTP

🤝 TCP 3-Way-Handshake

Verbindungsaufbau in drei Schritten:

  • Schritt 1 – SYN: Client sendet SYN-Paket (Seq=x)
  • Schritt 2 – SYN-ACK: Server antwortet mit SYN+ACK (Seq=y, Ack=x+1)
  • Schritt 3 – ACK: Client bestaetigtt mit ACK (Seq=x+1, Ack=y+1)

Verbindungsabbau: 4-Way – FIN, ACK, FIN, ACK

🗃 TCP-Header Felder

  • Source Port / Dest Port (je 16 Bit)
  • Sequenznummer (32 Bit) – Position der Daten
  • Bestätigungsnummer (32 Bit) – nächste erwartete Seq
  • Flags: SYN, ACK, FIN, RST, PSH, URG
  • Window Size (16 Bit) – Flusskontrolle
  • Checksum (16 Bit) – Fehlererkennung

🛣 RIP – Routing Information Protocol

  • Typ: Distance-Vector
  • Algorithmus: Bellman-Ford
  • Metrik: Hop Count (max. 15 Hops)
  • Updates: Alle 30 Sekunden (gesamte Routing-Tabelle)
  • Konvergenz: Langsam, anfaellig für Routing-Loops
  • Einsatz: Kleine Netzwerke

🌐 OSPF – Open Shortest Path First

  • Typ: Link-State
  • Algorithmus: Dijkstra (SPF)
  • Metrik: Kosten (basierend auf Bandbreite)
  • Areas: Hierarchisch mit Backbone-Area 0
  • Konvergenz: Schnell, loop-frei
  • Einsatz: Grosse Unternehmensnetzwerke (IGP)

🌎 BGP – Border Gateway Protocol

  • Typ: Path-Vector
  • Metrik: AS-Pfad, Policies
  • AS-Nummern: Jedes autonome System hat eine eindeutige ASN
  • Einsatz: Inter-Domain Routing (zwischen ISPs)
  • Läuft über: TCP Port 179
  • Konvergenz: Langsam, aber stabil und skalierbar

💡 Merke

TCP = zuverlässig, aber langsamer (Overhead durch Handshake, ACKs). UDP = schnell, aber ohne Garantie. Wahl haengt vom Anwendungsfall ab.

🎯 Prüfungstipp

Portnummern auswendig lernen! Häufige Frage: «Welches Protokoll nutzt welchen Port?» Merke: 80=HTTP, 443=HTTPS, 22=SSH, 53=DNS, 25=SMTP.

🧩 Routing-Vergleich

RIP = einfach (Distance-Vector, max 15 Hops). OSPF = schnelle Konvergenz (Link-State, Dijkstra). BGP = Internet-Backbone (Path-Vector, AS-Nummern).

🔒 Handshake merken

SYN → SYN-ACK → ACK – wie ein Telefongespräch: «Hallo?» – «Ja, hallo!» – «Gut, lass uns reden.»
