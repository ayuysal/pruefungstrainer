Abstrakter Datentyp (ADT)

Was vs. Wie

Was ist ein ADT?

Ein Abstrakter Datentyp beschreibt eine Datenstruktur rein über ihre Operationen und deren Verhalten – ohne festzulegen, wie sie intern implementiert wird. Er definiert das Was (Schnittstelle), nicht das Wie (Implementierung).

🔷 Spezifikation (ADT)

Definiert: Welche Operationen gibt es? Was sind Vor-/Nachbedingungen? Wie verhalten sie sich?

Beispiel: „Ein Stack hat push, pop, top, isEmpty"

🔶 Implementierung

Legt fest: Wie werden die Daten gespeichert? Array? Verkettete Liste? Baum?

Beispiel: „Stack mit Array" oder „Stack mit Linked List"

Stack

LIFO · LAST IN, FIRST OUT
Wie ein Bücherstapel: Das zuletzt abgelegte Element wird als erstes wieder entnommen. Zugriff nur oben (Top).
↑ TOP
BOTTOM
Bereit. Gib einen Wert ein und drücke push.

Operationen

push(x)Element oben ablegenO(1)
pop()Oberstes Element entfernen & zurückgebenO(1)
peek()/top()Oberstes Element ansehen (ohne Entfernen)O(1)
isEmpty()Ist der Stack leer?O(1)
size()Anzahl ElementeO(1)

Anwendungen

Undo/Redo in Editoren, Klammerprüfung (Compiler), Funktionsaufrufe (Call Stack), Tiefensuche (DFS), Postfix-Auswertung (Taschenrechner), Browser-Verlauf (Zurück-Button)
stack = [] stack.append("A") # push stack.append("B") stack.pop() # → "B"

Queue

FIFO · FIRST IN, FIRST OUT
Wie eine Warteschlange: Wer zuerst kommt, wird zuerst bedient. Einfügen hinten, Entnahme vorne.
← FRONT (raus)REAR (rein) →
Bereit. Gib einen Wert ein und drücke enqueue.

Operationen

enqueue(x)Element hinten anfügenO(1)
dequeue()Vorderstes Element entfernen & zurückgebenO(1)
front()/peek()Vorderstes Element ansehenO(1)
isEmpty()Ist die Queue leer?O(1)
size()Anzahl ElementeO(1)

Anwendungen

Druckerwarteschlange, CPU-Scheduling (Round Robin), Breitensuche (BFS), Puffer in Netzwerken (Router), Tastatureingabe-Puffer, Ticketsysteme
from collections import deque q = deque() q.append("A") # enqueue q.append("B") q.popleft() # → "A"

Verkettete Liste

LINKED LIST · Einfach verkettet
Elemente (Knoten) sind über Zeiger (Pointer) verbunden. Jeder Knoten kennt seinen Nachfolger. Einfügen/Löschen an beliebiger Stelle möglich.
HEAD →
Bereit. Füge Elemente per prepend oder append hinzu.

🧩 ADT vs. Implementierung

Stack und Queue sind ADTs – sie definieren Verhalten (LIFO/FIFO). Eine Linked List oder ein Array ist eine mögliche Implementierung dafür.

🎯 Prüfungstipp

Häufige Frage: „Zeige den Zustand nach push(3), push(5), pop(), push(7)". Nutze den Simulator oben, um solche Sequenzen durchzuspielen!

💡 Merkhilfe

Stack = Stapel (Teller) → LIFO.
Queue = Schlange (Kasse) → FIFO.
Liste = Kette → flexibel, aber langsam beim Suchen.
