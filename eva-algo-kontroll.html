Algorithmusbegriff

6 Eigenschaften

Ein Algorithmus ist eine endliche, eindeutige Handlungsvorschrift zur Lösung eines Problems. Er besteht aus einzelnen Schritten und muss bestimmte Eigenschaften erfüllen, um als Algorithmus zu gelten:

🔀

Kontrollstrukturen

Sequenz · Selektion · Iteration
▶ Sequenz
◆ Selektion
↻ Iteration

Sequenz

SEQUENCE · Anweisungsfolge
Anweisungen werden der Reihe nach von oben nach unten abgearbeitet – ohne Sprünge oder Verzweigungen. Jeder Schritt wird genau einmal ausgeführt.
// Sequenz-Beispiel print("Schritt 1") print("Schritt 2") print("Schritt 3")

Selektion

SELECTION · Verzweigung / Bedingte Anweisung
Der Programmablauf verzweigt basierend auf einer Bedingung. Ist sie wahr, wird ein Pfad ausgeführt, sonst ein anderer (if/else). Varianten: einseitig (nur if), zweiseitig (if/else), mehrseitig (switch/case).
// Selektion-Beispiel if alter >= 18: print("Volljährig") else: print("Minderjährig")

Iteration

ITERATION · Schleife / Wiederholung
Ein Codeblock wird wiederholt ausgeführt, solange (oder bis) eine Bedingung erfüllt ist. Typen: kopfgesteuert (while – Prüfung vor Durchlauf), fussgesteuert (do-while – mindestens 1× Durchlauf), Zählschleife (for – feste Anzahl).
// Iteration-Beispiele for i in range(5): print(i) # Zählschleife while x > 0: x = x - 1 # kopfgesteuert

🧩 Zusammenhang

Das EVA-Prinzip beschreibt den Gesamtfluss. Algorithmen bestimmen die Verarbeitungslogik (das „V"). Kontrollstrukturen steuern den Ablauf innerhalb eines Algorithmus.

🎯 Prüfungstipp

Typische Frage: „Nenne die 6 Eigenschaften eines Algorithmus." Merkhilfe: E-D-T-A-E-F – Endlichkeit, Determiniertheit, Terminierung, Allgemeingültigkeit, Eindeutigkeit, Finitheit.

📝 Darstellungsformen

Algorithmen können als Pseudocode, Struktogramm (Nassi-Shneiderman), Flussdiagramm oder in einer Programmiersprache dargestellt werden.
