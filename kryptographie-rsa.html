Referenz

🔒 Symmetrische Verschlüsselung

Ein gemeinsamer Schlüssel für Ver- und Entschlüsselung.

  • Beispiele: AES, DES, 3DES, Blowfish
  • Sehr schnell & effizient
  • Problem: Schlüsselverteilung – wie kommt der Schlüssel sicher zum Empfänger?
  • n Teilnehmer brauchen n(n-1)/2 Schlüssel

🔐 Asymmetrische Verschlüsselung

Öffentlicher Schlüssel (Public Key) + Privater Schlüssel (Private Key).

  • Beispiele: RSA, ECC, Diffie-Hellman
  • Kein Schlüsselverteilungsproblem
  • Deutlich langsamer als symmetrisch
  • Public Key verschlüsselt → nur Private Key entschlüsselt

🔀 Hybridverfahren

Kombination beider Verfahren für das Beste aus beiden Welten.

  • Asymmetrisch für Schlüsselaustausch
  • Symmetrisch für Nutzdaten
  • Beispiel: TLS/SSL (HTTPS)
  • Session Key wird per RSA ausgetauscht, Daten per AES verschlüsselt

🔢 RSA-Verfahren

  1. Wähle zwei Primzahlen p und q
  2. Berechne n = p × q
  3. Berechne φ(n) = (p−1)(q−1)
  4. Wähle e mit ggT(e, φ(n)) = 1
  5. Berechne d mit e × d ≡ 1 mod φ(n)
  6. Verschl.: c = me mod n
  7. Entschl.: m = cd mod n

Öffentlicher Schlüssel: (n, e) • Privater Schlüssel: (n, d)

✍️ Digitale Signatur

Sichert Authentizität und Integrität eines Dokuments.

  1. Hash des Dokuments berechnen (z.B. SHA-256)
  2. Hash mit privatem Schlüssel signieren
  3. Empfänger prüft Signatur mit öffentlichem Schlüssel
  4. Stimmt der entschlüsselte Hash überein? → Signatur gültig

📜 Zertifikate & PKI

Public Key Infrastructure verwaltet digitale Zertifikate.

  • Zertifikat bindet öffentlichen Schlüssel an Identität
  • Ausgestellt von einer CA (Certificate Authority)
  • X.509-Standard, enthält: Name, Public Key, Gültigkeit, Signatur der CA
  • Vertrauenskette: Root CA → Intermediate CA → Endnutzer

💡 Merke

RSA-Sicherheit basiert auf der Schwierigkeit, grosse Zahlen in ihre Primfaktoren zu zerlegen. In der Praxis werden Schlüssellängen von mindestens 2048 Bit verwendet.

🎯 Prüfungstipp

RSA-Berechnungen mit kleinen Primzahlen kommen fast immer in der Prüfung! Übe besonders:

  • φ(n) berechnen
  • d über den erweiterten Euklid finden
  • Modulare Potenz me mod n

🔒 Symmetrisch vs. Asymmetrisch

Symmetrisch = schnell, ein Schlüssel. Asymmetrisch = langsam, zwei Schlüssel. In der Praxis kombiniert man beide (Hybrid) – z.B. bei TLS/HTTPS.

✍️ Digitale Signatur

Signieren = Hash mit Private Key verschlüsseln. Verifizieren = mit Public Key entschlüsseln und Hash vergleichen. Wichtig: Signatur sichert Integrität UND Authentizität.
