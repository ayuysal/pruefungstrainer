Switch, Router, STP & VLAN

Netzwerkgeräte, Domains, Spanning Tree und VLANs verstehen und interaktiv üben

📚

Referenz

Hub vs Switch vs Router

  • Hub (Layer 1): Repeater, leitet Signale an ALLE Ports weiter. 1 Collision Domain, 1 Broadcast Domain.
  • Switch (Layer 2): Lernt MAC-Adressen, leitet gezielt weiter. Jeder Port = eigene Collision Domain, 1 Broadcast Domain.
  • Router (Layer 3): Verbindet Netze, trennt Broadcast Domains. Routing-Entscheidungen per IP.

Collision & Broadcast Domains

  • Collision Domain: Bereich, in dem Datenkollisionen auftreten können. Hub = 1 CD für alle Ports; Switch = 1 CD pro Port.
  • Broadcast Domain: Bereich, in dem ein Broadcast alle Geräte erreicht. Nur Router (oder VLAN) trennt Broadcast Domains.

Switch-Funktionen

  • MAC-Address-Table: Zuordnung MAC → Port, dynamisch gelernt
  • Store-and-Forward: Ganzes Frame empfangen, CRC prüfen, dann weiterleiten
  • Cut-Through: Weiterleitung nach Lesen der Ziel-MAC (schneller, keine Fehlerprüfung)
  • Learning: Quell-MAC merken | Flooding: Unbekanntes Ziel an alle | Forwarding: Bekanntes Ziel gezielt senden | Filtering: Frames im selben Segment nicht weiterleiten

Spanning Tree Protocol (STP)

  • Zweck: Schleifen in redundanten L2-Topologien verhindern
  • Root Bridge: Switch mit niedrigster Bridge Priority (Default 32768) + niedrigster MAC
  • BPDU: Bridge Protocol Data Units zum Austausch von STP-Infos
  • Port States: Blocking → Listening → Learning → Forwarding (+ Disabled)
  • Root Ports, Designated Ports, Non-Designated (Blocked) Ports

VLANs (Virtual LANs)

  • Logische Trennung auf Layer 2 – ein Switch, mehrere Broadcast Domains
  • Access Port: Gehört zu genau einem VLAN (untagged)
  • Trunk Port: Trägt mehrere VLANs (tagged, 802.1Q Header mit VLAN-ID)
  • 802.1Q: Standard für VLAN-Tagging, 4 Byte Tag im Ethernet-Frame
  • Native VLAN: Frames ohne Tag auf dem Trunk gehören zum Native VLAN

Router & Inter-VLAN-Routing

  • Layer 3: Routing-Entscheidung anhand IP-Adresse und Routing-Tabelle
  • Default Gateway: Standardroute für Pakete außerhalb des lokalen Netzes
  • NAT: Network Address Translation – private IP → öffentliche IP
  • PAT: Port Address Translation – viele private IPs teilen eine öffentliche IP (Portnummern)
  • Inter-VLAN-Routing: Router-on-a-Stick (Subinterfaces) oder Layer-3-Switch

Merke: Domains

Hub = 1 Collision Domain für alle. Switch = 1 Collision Domain PRO Port. Router trennt Broadcast Domains. Jedes Router-Interface = eigene Broadcast Domain.

Prüfungstipp: STP

Root Bridge = niedrigste Priority + niedrigste MAC. Alle Ports der Root Bridge sind Designated Ports. Auf jedem Nicht-Root-Switch gibt es genau einen Root Port (bester Pfad zur Root Bridge).

Prüfungstipp: VLAN

Access Port = 1 VLAN (untagged). Trunk = mehrere VLANs (tagged mit 802.1Q). Ohne Router/L3-Switch können VLANs nicht miteinander kommunizieren!

Eselsbrücke

Hub = dumm (alles an alle). Switch = schlau (MAC-Tabelle). Router = am schlausten (IP-Routing, trennt Broadcasts). Merke: Je höher der Layer, desto intelligenter das Gerät.
