Anwendungsprotokolle

HTTP, DNS, E-Mail, DHCP, FTP, SSH – Protokolle verstehen und Portnummern sicher beherrschen

📚

Referenz

🌐 HTTP / HTTPS

  • Ports: 80 (HTTP), 443 (HTTPS/TLS)
  • Methoden: GET (lesen), POST (senden), PUT (ersetzen), DELETE (löschen)
  • Statuscodes: 1xx Info, 2xx Erfolg, 3xx Umleitung, 4xx Client-Fehler, 5xx Server-Fehler
  • Aufbau: Request-Line, Header, Body – Response: Statuszeile, Header, Body
  • HTTP/2: Multiplexing, Header-Kompression, binär statt Text

🔍 DNS

  • Port: 53 (UDP + TCP)
  • Hierarchie: Root (.) → TLD (.de, .com) → Authoritative NS
  • Auflösung: Rekursiv (Resolver erledigt alles) vs. Iterativ (schrittweise Verweise)
  • Records: A (IPv4), AAAA (IPv6), MX (Mail), CNAME (Alias), NS (Nameserver)

📧 E-Mail-Protokolle

  • SMTP: Port 25/587 – Versand von E-Mails
  • POP3: Port 110 – Abholen, löscht vom Server
  • IMAP: Port 143 – Synchronisation, bleibt auf Server
  • POP3 vs IMAP: POP3 = ein Gerät, offline; IMAP = mehrere Geräte, synchron

📌 DHCP

  • Ports: 67 (Server), 68 (Client)
  • DORA-Prozess: Discover → Offer → Request → Acknowledge
  • Lease: IP-Adresse wird zeitlich begrenzt vergeben
  • Client sendet Broadcast, Server antwortet mit IP-Angebot

📁 FTP

  • Port 21: Steuerungskanal (Befehle)
  • Port 20: Datenkanal (Dateiübertragung)
  • Aktiv: Server baut Datenverbindung zum Client auf
  • Passiv: Client baut beide Verbindungen auf (Firewall-freundlich)

🔒 Telnet vs SSH

  • Telnet: Port 23 – Klartext, unsicher, veraltet
  • SSH: Port 22 – Verschlüsselt, sicher, Standard für Remote-Zugriff
  • SSH unterstützt Authentifizierung per Passwort oder Public Key

💡 Merke

Well-Known Ports liegen im Bereich 0–1023. Prüfungsfragen betreffen fast immer diese Ports. HTTPS = HTTP + TLS auf Port 443.

🎯 Prüfungstipp

DHCP DORA-Reihenfolge wird häufig abgefragt! Merkhilfe: «Darf Olaf Rosinen Anbieten?» – Discover, Offer, Request, Acknowledge.

🧩 POP3 vs IMAP

POP3 lädt herunter und löscht vom Server – ideal für ein Gerät. IMAP synchronisiert – ideal für mehrere Geräte. Beide nur zum Empfang, Versand immer per SMTP.
